# 北京小客车家庭积分计算器 - 开发规范

## 发布流程

每次功能改动后，按以下顺序执行：

1. **构建验证**
   ```bash
   npm run build
   ```
   要求：构建成功（允许 TypeScript 版本警告）

2. **推送代码**
   ```bash
   git add .
   git commit -m "描述信息"
   git push origin main
   ```

3. **部署到 Cloudflare Pages**
   ```bash
   npx wrangler pages deploy out --project-name bj-car-points --branch main
   ```

## 官方政策规则（必须严格遵守）

参考：https://xkczb.jtw.beijing.gov.cn/bszn/20201230/1609342087846_1.html

### 一、个人积分计算

**家庭申请人积分 = 基础积分 + 阶梯（轮候）积分**

#### 1. 基础积分
- 家庭主申请人：2分
- 其他家庭申请人：每人1分

#### 2. 阶梯（轮候）积分

**2.1 普通摇号阶梯积分**

按累积的阶梯数，每1阶梯加1分。

**2020年12月31日前规则：**
- 0次 → 0分
- 1-6次 → 1分
- 7-12次 → 2分
- 13-18次 → 3分
- 19-24次 → 4分
- 25-30次 → 5分
- 31-36次 → 6分
- 37-42次 → 7分
- 43-48次 → 8分
- 49-54次 → 9分
- 55-60次 → 10分
- 61-66次 → 11分
- 67-72次 → 12分
- 73-78次 → 13分

**2021年1月1日后规则：**
- 每2次摇号增加1个阶梯分（上不封顶）
- 1-2次 → 1分
- 3-4次 → 2分
- 5-6次 → 3分
- 7-8次 → 4分
- 9-10次 → 5分
- ...以此类推

**说明**：2021年政策出台时，表格只列到25次（因为当时最多只能参加这么多次）。实际规则是每2次增加1分，持续累加，没有上限。

**特殊加分**：具有C5驾照的家庭主申请人，额外增加1个阶梯数（仅主申请人，且需参与普通摇号）

**2.2 新能源轮候积分**

按最近一次开始轮候的时间距离家庭摇号申请年上一年12月31日，每满一年加1分。

以往参加摇号获得的阶梯数与轮候年限分合并求和。

**重要说明：申请策略**

家庭申请时可以选择：
1. **仅普通指标**：只参加普通摇号（6月、12月）
2. **新能源指标或普通指标**（推荐）：
   - 5月26日先参加新能源指标配置（按积分排序）
   - 如果入围：获得新能源指标
   - 如果未入围：自动转入普通指标池，参加6月、12月的摇号
   - **优势**：新能源轮候时间持续累积，同时保留普通摇号机会
   - **积分计算**：普通摇号阶梯分 + 新能源轮候年限分（两者同时累积）

**注意**：如果中途从"新能源指标或普通指标"改为"仅普通指标"，新能源轮候时间会中断重置。建议一旦开始轮候新能源，就持续选择"新能源指标或普通指标"。

#### 3. 家庭申请年限加分

以家庭为单位申请，每满一年，**所有家庭申请人积分各增加1分**。

### 二、家庭总积分计算

#### 含配偶
```
总积分 = [(主申请人积分 + 配偶积分) × 2 + 其他成员积分之和] × 家庭代际数
```

#### 不含配偶
```
总积分 = (主申请人积分 + 其他成员积分之和) × 家庭代际数
```

**家庭代际数**：指家庭申请人中包含几代人，最多为3代。

### 三、计算示例

**示例：夫妻 + 1名子女（2代）**

假设：
- 家庭申请年限：3年
- 主申请人：从2020年开始摇号（2020年6次 + 2021-2025年10次 = 16次）
- 配偶：从2021年开始摇号（2021-2025年10次）
- 子女：从2023年开始轮候新能源（2年）

计算过程：
1. 主申请人积分 = 2（基础）+ 1（2020年前6次）+ 5（2021年后10次，10÷2=5）+ 3（家庭年限）= 11分
2. 配偶积分 = 1（基础）+ 5（2021年后10次，10÷2=5）+ 3（家庭年限）= 9分
3. 子女积分 = 1（基础）+ 2（轮候2年）+ 3（家庭年限）= 6分
4. 家庭总积分 = [(11 + 9) × 2 + 6] × 2 = 92分

## 产品实现要求

1. **不允许用户手填最终分值**，必须通过输入次数/年限自动计算
2. **必须展示计算过程**，让用户看到每一步的分数来源
3. **必须显示免责声明**："以当年官方公告和系统结果为准"
4. **UI要求**：
   - 清晰的步骤流程
   - 实时计算预览
   - 详细的计算明细
   - 响应式设计
   - 美观的视觉效果
